2
硬體腳位與型號提醒（三針腳 vs 四針腳）

⚠️ 型號與電阻關鍵確認：

  • 3 腳位模組（常見藍色小板）： 通常已內建上拉電阻，因此通常不需外接電阻。接線時只需將模組的 VCC、GND、DATA 依序連接樹莓派即可。
  • 4 腳位元件（裸件）： 內部沒有整合電阻，必須在 VCC 與 DATA 之間手動外接一個 4.7kΩ ~ 10kΩ 的上拉電阻，否則樹莓派可能因訊號浮動而無法正確讀取到溫濕度數據（第 3 腳通常為空腳不接）。

Raspberry Pi 40-pin 實體排針接法對照表（樹莓派 3）：

左側功能標記 (奇數針腳) 實體 實體 右側功能標記 (偶數針腳)
接 DHT11 的 VCC（標示 + 或 VCC） ── [3.3V 電源 / Pin 1] 1 2 5V 電源
GPIO 2 (SDA) 3 4 5V 電源
GPIO 3 (SCL) 5 6 接 DHT11 的 GND（標示 - 或 GND） ── [Ground 地線 / Pin 6]
接 DHT11 的 DATA（標示 S、OUT 或 DAT） ── [GPIO 4 / Pin 7] 7 8 GPIO 14 (TXD)
Ground (接地) 9 10 GPIO 15 (TXD)

樹莓派本地中繼與直連 LINE Bot 控制主程式
import time
import requests
import board
import adafruit_dht

# ==================== 1. 參數與憑證設定 ====================
SPREADSHEET_ID = "貼上你檢索到的Google試算表ID亂碼"
LINE_ACCESS_TOKEN = "請在此處貼上你的_LINE_Channel_Access_Token"
TARGET_USER_ID = "請在此處貼上你的_LINE_Your_user_ID"

dhtDevice = adafruit_dht.DHT11(board.D4)

def send_to_line_bot_direct(temp, humi):
    """樹莓派直接呼叫 LINE 官方主機 API 推播"""
    line_api_url = "https://api.line.me/v2/bot/message/push"
    current_time = time.strftime("%Y-%m-%d %H:%M:%S", time.localtime())
    
    message_text = f"\n🌡️ 樹莓派溫溼度直連回報\n-------------------\n⏰ 時間：{current_time}\n°C 溫度：{temp} °C\n💧 濕度：{humi} %"
    
    headers = {
        "Content-Type": "application/json",
        "Authorization": f"Bearer {LINE_ACCESS_TOKEN}"
    }
    payload = {
        "to": TARGET_USER_ID,
        "messages": [{"type": "text", "text": message_text}]
    }
    try:
        response = requests.post(line_api_url, json=payload, headers=headers, timeout=10)
        if response.status_code == 200:
            print("【LINE Bot 直連】手機推播成功！")
        else:
            print(f"【LINE Bot】發送失敗: {response.status_code}")
    except Exception as e:
        print(f"【LINE Bot】通訊異常: {e}")
