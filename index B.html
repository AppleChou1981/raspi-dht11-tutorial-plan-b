DHT11 溫溼度數據收集：樹莓派本地端中繼傳送方案 (方案 B)

💡 架構導讀（方案 B）： 本教學採用「方案 B」。樹莓派作為本地中繼站與發起核心，讀取 DHT11 後，直接在樹莓派上執行高級驗證，透過 Google Sheets API V4 安全金鑰 (JSON 憑證) 直接操作雲端資料表，並平行呼叫 LINE 伺服器。此方案不依賴 GAS 網頁部署，具備高度隱私權管理、安全性強且完全符合產業標準認證的特性。

階段一：Google Cloud Platform (GCP) 金鑰與權限設定

1
啟用 Google Sheets API
  1. 前往並登入 Google Cloud 建立主控台。
  2. 新增一個專案（例如命名為：`Raspi-Data-Center`）。
  3. 點擊左上角選單，進入 「API 和服務」 > 「庫」。
  4. 在搜尋欄輸入 「Google Sheets API」 並點擊進入，接著按下 「啟用」 按鈕。
2
建立憑證與下載 JSON 金鑰
  1. 啟用 API 後，點擊左側 「憑證」 頁籤，再點擊上方的 「+ 建立憑證」 > 選擇「服務帳戶 (Service Account)」。
  2. 填寫服務帳戶名稱，一路按下「建立並繼續」直至完成。
  3. 建立成功後，在憑證清單下方會看到一組外觀類似 Email 的「服務帳戶電子郵件位址」，請將它複製下來。
  4. 點擊該服務帳戶右側的 「編輯 (鉛筆圖示)」 > 切換到「金鑰」頁籤 > 點擊「新增金鑰」 > 選擇「建立新金鑰」。
  5. 規格務必選擇 JSON 格式，點擊建立。此時瀏覽器會自動下載一個 `.json` 格式的加密金鑰檔案，請妥善保存，切勿外流！
3
將試算表權限共享給服務帳戶
  1. 去你的 Google 雲端硬碟建立一張全新的 Google 試算表，並複製網址中的一長串亂碼（此即為 **Spreadsheet ID**）。
  2. 點擊該試算表右上角的 「共用」 按鈕。
  3. 在新增人員欄位中，貼上剛剛在 GCP 複製的 **「服務帳戶電子郵件位址」**，權限設定為 **「編輯者」**，取消勾選通知，並按下儲存。

階段二：LINE Developer 開發者憑證獲取

4
撈取 LINE 關鍵憑證字串

登入 LINE Developers 控制台 建立你的 Bot 機器人：

  • 獲取 User ID：在 Basic settings 頁籤最底部，複製 Your user ID（格式為 `Uxxxxxxxx...`）。
  • 獲取 Access Token：在 Messaging API 頁籤最底部，在 Channel access token 欄位點擊 「Issue」，複製整串長金鑰。
  • 加好友：用手機掃描該頁面的 QR Code，將機器人加入好友。

階段三：樹莓派 Python 環境建置與中繼程式撰寫

5
安裝 API 通訊與硬體驅動套件

由於方案 B 是在樹莓派本地端直接對 Google 伺服器進行認證，因此必須安裝專門的 Python 庫：

sudo apt update
sudo apt install -y gpiod libgpiod-dev
pip3 install gspread oauth2client adafruit-circuitpython-dht --break-system-packages
💡 小提示：請將剛剛下載的 Google `.json` 金鑰檔案傳入樹莓派中，並重新命名為更簡短好記的檔案名稱（例如：`creds.json`），放置在與你的 Python 腳本同一個資料夾底下。
