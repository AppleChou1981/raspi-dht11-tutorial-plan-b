DHT11 溫溼度數據收集：樹莓派本地端中繼傳送方案 (方案 B)

💡 架構導讀（方案 B）： 本教學採用「方案 B」。樹莓派作為本地中繼站與發起核心，讀取 DHT11 後，直接在樹莓派上執行高級驗證，透過 Google Sheets API V4 安全金鑰 (JSON 憑證) 直接操作雲端資料表，並平行呼叫 LINE 官方伺服器。此方案不依賴 GAS 網頁應用程式部署，具備高度隱私權管理、安全性強且完全符合業界標準 OAuth2 認證的特性。

階段一：Google Cloud Platform (GCP) 金鑰與權限設定

1
啟用 Google Sheets API
  1. 前往並登入 Google Cloud 控制台。
  2. 新增一個專案（例如命名為：`Raspi-Data-Center`）。
  3. 點擊左上角選單，進入 「API 和服務」 > 「庫」。
  4. 在搜尋欄輸入 「Google Sheets API」 並點擊進入，接著按下 「啟用」 按鈕。
2
建立憑證與下載 JSON 金鑰
  1. 啟用 API 後，點擊左側 「憑證」 頁籤，再點擊上方的 「+ 建立憑證」 > 選擇「服務帳戶 (Service Account)」。
  2. 填寫服務帳戶名稱，一路按下「建立並繼續」直至完成。
  3. 建立成功後，在憑證清單下方會看到一組外觀類似 Email 的「服務帳戶電子郵件位址」，請將它完整複製下來。
  4. 點擊該服務帳戶右側的 「管理金鑰」 或點擊進入後選擇 「金鑰」頁籤 > 點擊「新增金鑰」 > 選擇「建立新金鑰」。
  5. 規格務必選擇 JSON 格式，點擊建立。此時瀏覽器會自動下載一個 `.json` 格式的加密金鑰檔案，請妥善保存，切勿外流！
3
將試算表權限共享給服務帳戶
  1. 去你的 Google 雲端硬碟建立一張全新的 Google 試算表，並複製網址中的一長串亂碼（此即為 **Spreadsheet ID**）。
  2. 點擊該試算表右上角的 「共用」 按鈕。
  3. 在新增人員欄位中，貼上剛剛在 GCP 複製的 **「服務帳戶電子郵件位址」**。
  4. 權限設定為 **「編輯者」**，取消勾選通知，並按下儲存。這樣樹莓派才有權限直接改動這張表。

階段二：LINE Developer 開發者憑證獲取

4
撈取 LINE 關鍵憑證字串

登入 LINE Developers 控制台 建立你的 Bot 機器人：

  • 獲取 User ID：在 Basic settings 頁籤最底部，複製 Your user ID（格式為 `Uxxxxxxxx...`）。
  • 獲取 Access Token：在 Messaging API 頁籤最底部，在 Channel access token 欄位點擊 「Issue」，複製整串長金鑰。
  • 加好友：用手機掃描該頁面的 QR Code，將機器人加入好友，否則會收不到主動推播。

階段三：樹莓派 Python 環境建置與中繼程式撰寫

5
安裝 API 通訊與硬體驅動套件

由於方案 B 是在樹莓派本地端直接對 Google 官方伺服器進行 OAuth2 加密認證，因此必須安裝專門的 Python 通訊庫：

sudo apt update
sudo apt install -y gpiod libgpiod-dev
pip3 install gspread oauth2client adafruit-circuitpython-dht --break-system-packages
💡 檔案放置小提示：請將剛剛在電腦上下載的 Google GCP `.json` 金鑰檔案透過傳輸軟體傳入樹莓派中，並重新命名為更簡短好記的檔案名稱（例如：`creds.json`），將其放置在與你的 Python 腳本同一個資料夾目錄底下。
6
硬體引腳彈性對接

請先用肉眼認清你手上的 3-Pin DHT11 引腳上的文字標示（VCC、GND、Data），並在樹莓派關機狀態下一對一安全連線：

DHT11 模組上的標示 連接到樹莓派的腳位名稱 樹莓派實體排針編號 (Physical Pin)
VCC / + (電源正極) 3.3V 電源 實體第 1 針腳 (Pin 1)
GND / - (電源負極) GND (Ground 地線) 實體第 6 針腳 (Pin 6)
DATA / S / OUT (數據腳) GPIO 4 實體第 7 針腳 (Pin 7)
7
撰寫 Python 本地中繼控制主程式

在樹莓派終端機輸入 nano dht11_local_relay.py，並貼入以下融合了本地安全授權與 LINE Messaging API 的完整程式碼：

import time
import requests
import board
import adafruit_dht
import gspread
from oauth2client.service_account import ServiceAccountCredentials

# ==================== 1. 雲端與金鑰引腳參數設定 ====================

# (1) Google Sheets 參數設定
JSON_KEY_FILE = "creds.json"  # 請確保與此 Python 檔同目錄
SPREADSHEET_ID = "請在此處替換為你的_Google_試算表網址中的長串ID"

# (2) LINE Bot 憑證參數設定
LINE_ACCESS_TOKEN = "請在此處貼上你的_LINE_Channel_Access_Token"
TARGET_USER_ID = "請在此處貼上你的_LINE_Your_user_ID"

# (3) 硬體引腳預設 GPIO 4 (實體第 7 針)
dhtDevice = adafruit_dht.DHT11(board.D4)

# ==================== 2. 功能函式定義 ====================

def send_to_google_sheet_direct(temp, humi):
    """利用本機端憑證，透過 Google Sheets API 直接授權寫入試算表"""
    scope = ["https://spreadsheets.google.com/feeds", "https://www.googleapis.com/auth/drive"]
    try:
        creds = ServiceAccountCredentials.from_json_keyfile_name(JSON_KEY_FILE, scope)
        client = gspread.authorize(creds)
        
        # 透過 ID 開啟指定的試算表
        sheet = client.open_by_key(SPREADSHEET_ID).sheet1
        
        now = time.strftime("%Y-%m-%d %H:%M:%S", time.localtime())
        sheet.append_row([now, temp, humi])
        print("【Google Sheets API】利用本機授權直接寫入成功！")
    except Exception as e:
        print(f"【Google Sheets API】認證或寫入發生錯誤: {e}")

def send_to_line_bot_direct(temp, humi):
    """從樹莓派直接呼叫 LINE Messaging API 推送通知"""
    url = "https://api.line.me/v2/bot/message/push"
    current_time = time.strftime("%Y-%m-%d %H:%M:%S", time.localtime())
    
    message_text = f"\n🌡️ 樹莓派本地中繼端回報 (方案B)\n-------------------\n⏰ 時間：{current_time}\n°C 溫度：{temp} °C\n💧 濕度：{humi} %"
    
    headers = {
        "Content-Type": "application/json",
        "Authorization": f"Bearer {LINE_ACCESS_TOKEN}"
    }
    payload = {
        "to": TARGET_USER_ID,
        "messages": [{"type": "text", "text": message_text}]
    }
    try:
        response = requests.post(url, json=payload, headers=headers, timeout=10)
        if response.status_code == 200:
            print("【LINE Bot】訊息直發推送成功！")
        else:
            print(f"【LINE Bot】發送失敗，狀態碼: {response.status_code}")
    except Exception as e:
        print(f"【LINE Bot】發送連線異常: {e}")

# ==================== 3. 主程式執行迴圈 ====================
if __name__ == "__main__":
    print("樹莓派 方案 B (本地中繼安全雙向系統) 啟動...")
    try:
        while True:
            try:
                temperature = dhtDevice.temperature
                humidity = dhtDevice.humidity
                
                if humidity is not None and temperature is not None:
                    temp_fixed = round(temperature, 1)
                    humi_fixed = round(humidity, 1)
                    
                    print(f"\n[偵測數據] 溫度: {temp_fixed}°C | 濕度: {humi_fixed}%")
                    
                    # 依序執行本地雙發任務
                    send_to_google_sheet_direct(temp_fixed, humi_fixed)
                    send_to_line_bot_direct(temp_fixed, humi_fixed)
                
                # 每 10 分鐘自動讀取發送一次
                time.sleep(600)

            except RuntimeError as error:
                # DHT11 對時序敏感，偶發讀取失敗為正常物理特性，等待2秒重試
                print(f"讀取中發生微小錯誤 (重試中): {error.args[0]}")
                time.sleep(2.0)
                continue
            except Exception as error:
                dhtDevice.exit()
                raise error
    except KeyboardInterrupt:
        dhtDevice.exit()
        print("\n中繼程式已由使用者手動終止。")

替換參數完畢後，按 Ctrl + O 儲存，再按 Ctrl + X 離開編輯器。

8
專案啟動與驗證

在樹莓派終端機執行下方指令啟動系統：

python3 dht11_local_relay.py

💡 成果驗證：此時樹莓派會調用本機的 `creds.json` 憑證與 Google 伺服器進行安全握手。只要終端機顯示寫入成功，你就可以到 Google 試算表與手機 LINE 進行最後的對接驗證了！
